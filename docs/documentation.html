Documentation

In this page, I will explain how to use my design system properly.


Colours

First is the primary colours. the classes structure is primary- and then the numbers 100, 200,300,400,500 and 600.
100 is the brighests while 600 is the darkest. For example "primary-300". Use this as the main colour in your design.

Second is the secondary colours. the classes structure is secondary- and then the numbers 100, 200,300,400,500 and 600.
100 is the brighests while 600 is the darkest. For example "secondary-300". Use this as the side colour in your design.

Third is the neutral colours. the classes structure is neutral- and then the numbers 100, 200,300,400,500 and 600.
100 is the brighests while 600 is the darkest. For example "neutral-300". Use this as the main colour in your design.

Next we several miscellaneous colours.
The class warning-colour is used for warnings and actions that prompt them.
The Class link-colour is used for links.
The class dark-text-colour is used for text on a light background.
The class light-text-colour is used for text on a dark background.
The class accessible-warning-colour is used in place of warning-colour in a colour blind accessible design.



Typography

First to use headings, simply use the classes heading-1, heading-2 or heading-3.
Please use heading-1 for titles, heading-2 for section titles within a page and heading-3 for titles to paragraphs.

Second is the body. use the classes body-1, body-2, body-3. body-1 is the largest, while body-3 is the smallest.
Use these classes as you see fit for your primary text.

Third is bold. use the classes bold-1, bold-2, and bold-3. As with body, bold-1 is the largest and bold-3 is the smallest.
Use these classes Labels and highlighted text.

And finally fourth are links. Use the classes link-1, link-2 and link-3. As with the other classes, link-1 is the biggest and link-3 is the smallest.
Use these classes for links only.


Buttons

The button classes are to be used on buttons only.
The button classes are structure as so, button-size-attribute. The Sizes are lg=large, md=medium and sm=small.
For Example "button-lg".
attributes are optional, and are -highlight and -disabled. Use the highlight attribute when you want to highlight a button as more important.
Use the disabled attribute for when the button is not suppose to function.
For example "button-lg-disabled".
To make a delete button, put del- before the button, like so "del-button-lg".
To make an accessible del button, put del-ac- before the button, like so "del-ac-button-lg".

Messages

The message classes are used for messages to the user.
warning-message is used for waring messages.
success-message is used for success messages.
loading-message is used when something is loading.
warning-ac-message is used in place of warning-message, in a colour blind accessible design.


Inputs

The input classes are use on inputs or textareas.
Use the class "input" on basic inputs.
use the class "textarea" on textareas.


Icons

The icons classes are used on i.
the icons are structured as icon- and then the name of the icon.
The names are profile, home, arrow, lock, search, settings, save, image, cancel, bin, filter, tickmark and plus.
For example "icon-filter".
Add -2 at the end of purple icons to make them green.
Use icon-bin-ac in place of icon-bin, in a colour blind accessible design.


Selectors

Selectors require a specific html structure to work.
For the Check mark use this structure below.
<label class="concheck">
<input class="checkbox" type="checkbox" name="checkbox">
<span class="checkmark"> </span>
<label>

For the radion button use this structure below.
<label class="Radio">
<input class="radioBox" type="checkbox" name="checkbox">
<span class="radioMark"></span>
</label>

For the switch button use the structure below.
<label class="switch">
<input type="checkbox">
<span class="slider"></span>
</label>
